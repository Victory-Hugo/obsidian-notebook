软件介绍

PAML（Phylogenetic Analysis by Maximum Likelihood）是一款广泛使用的生物信息学软件，主要用于基于最大似然方法的分子序列数据的进化分析。这个工具主要用来估计核苷酸或氨基酸序列之间的进化关系，如分支长度的估计。PAML 特别适用于复杂的统计模型，如可变替换率和不同的进化模型。

请注意，这个软件不适用于大规模数据的最大似然树建立!!!

相反的，你应该需要使用其它建树软件建立最大似然树，并连同 树文件 和 序列文件 一同输入到 PAMLX 软件中进行计算。
建树软件 iqtree：线粒体最大似然树 等

软件使用

BaseML 方法

输入部分

  • seqfile：指定序列数据文件的路径。
  • treefile：指定进化树文件的路径。

输出部分

  • outfile name：输出文件的名称。
  • location：输出文件的保存位置。

参数配置

  • fix k：固定转换/颠换比率（κ）的值。如果选中，用户必须提供κ的值。请阅读图解核苷酸替换模型（Nucleotide substitution models）
    • fix_kappa 用于指定是否固定 K80、F84 或 HKY85 模型中的转换与颠换比率参数 κ。
    • 如果 fix_kappa = 1，则 kappa 的值被固定；
    • 如果 fix_kappa = 0，则 kappa 的值用作迭代的初始估计值。
    • 对于 JC69 或 F81（没有 κ 参数）以及 TN93 和 REV（分别有两个和五个速率参数，且都从数据中估计）模型，fix_kappa 和 kappa 无效。
    • fix_kappa = 2 与 nhomo = 5 一起用于 baseml 中指定非平稳模型，其中整个速率矩阵 Q（包括交换性参数和碱基频率参数）在不同分支上变化。
  • fix α：固定 gamma 分布形状参数（α）。如果选中，α的值为0，表示不考虑位点间的变异。
    • fix_alpha 和 alpha 选项控制跨位点的可变替换速率的 gamma 分布的形状参数 α。当选中了 fix_alpha 且alpha = 0（0 表示无穷大）时，所有位点的替换率相同。正值的 alpha 表示使用离散 gamma 模型，ncatG 是离散 gamma 模型的类别数。
  • different alpha's for genes：为不同基因设置不同的 alpha 值。
  • ncatG：离散 gamma 模型的类别数。
  • fix ρ：固定相关性参数ρ。选中时，ρ的值为0，表示位点间的替换率独立。
  • TipDate：如果进行时间相关的序列分析，此处可以输入样本的采集时间。
  • time unit：时间单位，用于时间树分析。
  • nparK：选择非参数速率模型的类型。
  • Small_Diff：算法中使用的数值微分阈值。
  • ndata：数据集的数量，用于多数据集分析。
    • ndata 设置ndata 变量用于指定序列文件中数据集的数量，有助于处理模拟数据集或多基因对齐数据。具体设置如下：
    • ndata = 100：所有数据集/对齐使用相同的树结构（一个树块）。
    • ndata = 100 separate_trees：每个对齐有其自己的树块。
    • ndata = 100 maintree：使用主树文件生成每个基因对齐的子树，并进行最大似然（ML）分析。
    • ndata = 100 maintree 1：同上，进行ML分析。
    • ndata = 100 maintree 0：生成基因树文件，但不运行ML分析。
    • 我们一般选择默认。

运行模式和模型设置

  • runmode：选择树的使用方式，0 表示使用用户提供的树。
    • runmode = 0：评估树结构文件中指定的树拓扑。
    • runmode = 1 和 2：通过启发式搜索算法构建进化树，其中1从多分枝树开始，2从星形树开始。
    • runmode = 3：逐步添加法。
    • runmode = 4 和 5：从给定或计算的树开始进行NNI（最近邻互换）扰动。
    • 不建议对大规模数据集执行搜树，因为该软件的速度不能满足需求！
  • model：选择核苷酸替换模型，7 表示使用 REV（GTR）模型。
    • 模型编号：0 至 8 对应不同的替换模型，如 JC69、K80、F81、HKY85 等。
    • model = 9 和 10：分别对应特殊情况的 REV/GTR 和 UNREST 模型。
    • 自定义速率模型：如 UNRESTu 和 REVu，允许用户指定具有相同速率的核苷酸对。
    • 具体参见图解核苷酸替换模型（Nucleotide substitution models）
  • clock：选择分子钟模型，0 表示不使用分子钟。
    • clock 变量定义了分子钟假设的不同模型：
    • clock = 0：无分子钟模型，分支速率完全自由变化，使用无根树。
    • clock = 1：全局分子钟模型，所有分支速率相同，适用于有根树。
    • clock = 2：局部分子钟模型，大多数分支符合钟模型，默认速率为1，但某些特定分支可有不同速率。
    • clock = 3：用于多基因或多区段数据的分析，允许不同数据区段的分支速率以不同方式变化。
    • 对于人类的 Y 染色体和 mtDNA 来说，我们一般选择严格分子钟，在这里，也就是说我们选择1.另外，我们需要注意，如果我们输入的 DNA 中明显分区，那么我们就应该选择3.
  • Mgene：选择是否对基因或数据分区进行不同处理，0 表示速率相同。
    • 多基因或多位点分析（Mgene）Mgene = 0：未使用 G 选项，假设完全同质。
    • Mgene = 1：分区模型，对每个基因或位点使用相同的模型。
    • Mgene = 2、3、4：不同的设置允许基因之间具有不同的频率和速率参数。
    • 这点比较适合一段 DNA 序列包含多个分区的情况！
  • nhomo：选择是否使用非同质性模型，0 表示同质模型。
    • nhomo = 0：基本替换模型，碱基频率参数通过观察频率平均值估计。
    • nhomo = 1：同质模型，通过最大似然迭代估计频率参数 πT, πC 和 πA。
    • nhomo = 2：每个树枝使用一个转换/颠换比率（κ），适用于 K80、F84 和 HKY85 模型。
    • nhomo = 3, 4, 5：非同质模型，碱基组成在树上发生漂移，不同树枝使用不同的频率参数。
    • nhomo = 3：每个末端分支、所有内部分支和根使用各自的一套频率参数。
    • nhomo = 4：为根和每个分支设定不同的初始和末端频率参数。
    • nhomo = 5：用户指定多套频率参数，并定义每个分支使用哪套参数。
  • fix blength：选择是否固定分支长度。
  • optimization method：选择参数优化方法。
  • icode：选择遗传密码表，用于编码序列。

其他选项

  • getSE：计算参数的标准误差。
    • 建议勾选
  • RateAncestor：计算祖先节点的速率。
    • 可以勾选。即使勾选，也可以在结果中忽视。
  • clean data：在分析前清洗数据。
    • 可以勾选。

让我们来看看别人如何使用。PAML 软件经常被用来计算最近共同祖先时间(Time to most recent common ancestor，MRCA) 。怎么计算呢？
首先，利用建树软件获得最大似然树（例如 RaxML-NG、iqtree：基础操作等），然后将序列与树文件丢给 PAML 计算最大似然值。获得的最大似然值可以被当作是遗传距离的一种指标。

In details, we used PAMLX v1.3.1 (Yang 1997) to calculate maximum-likelihood (ML) estimates, assuming the HKY85 mutation model with gamma-distributed (32 categories) rates (plus invariant sites) and two partitions: coding region (positions 00577–16023) and control region (positions 16024–00576), as described elsewhere (Achilli et al. 2013). We converted ML mutational distances into years by employing both the corrected molecular clock embedded in the calculator of (Soares et al. 2009) and the linear mutation rate recently published of 2.7±0.2 x 10-8 base substitution per nucleotide per year (Posth et al. 2016)[1].

由上我们可知，该研究者使用该软件。选定了 HKY85 模型，并设定了32种突变速率。
