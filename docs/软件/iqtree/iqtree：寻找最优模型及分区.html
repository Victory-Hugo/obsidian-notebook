准备工作

比对序列

我们需要对原始的 fasta 文件进行对齐，因为在建立系统发育树必须使用对齐的文件。我们推荐使用 MAFFT 软件进行操作。

安装 MAFFT

sudo apt-get update
sudo apt-get install mafft

对齐

mafft --auto input.fasta > output_aligned.fasta

注意一下，我们使用的是自动策略，可能对精确度有一定的影响，我们在这里不展开叙述。
如果为了寻求更加精确的对齐，请参考如下教程。

  1. GUIDANCE2 Server ：多序列比对置信软件。
  2. MAFFT：多序列比对教程。
    总之，多序列比对，还是推荐MAFFT，基因编码区的多序列比对，更加推荐使用MAFFT基于密码子的方式比对。

切割序列

mtDNA 不同的区域的遵循不同的进化模型，因此，我们需要将不同的区域切割出来，寻找每个区域对应的最优模型，然后再进行 BEAST。

得到比对完成之后的 aln.fasta 之后，就可以进行切割了。 在这里，我做了一个 python，可以交互式切割指定的序列。

from Bio import SeqIO

def print_sequence_lengths(sequences):
    """打印每个序列的长度"""
    for record in sequences:
        print(f"ID: {record.id}, Length: {len(record.seq)}")

def get_ranges_from_user():
    """交互式获取用户输入的范围"""
    ranges = []
    while True:
        user_input = input("Enter range (e.g., 1-200) or 'OK' to finish: ")
        if user_input.strip().upper() == 'OK':
            break
        try:
            start, end = map(int, user_input.split('-'))
            ranges.append((start, end))
        except ValueError:
            print("Invalid range. Please enter again.")
    return ranges

def split_fasta(input_file, sequences, ranges):
    """
    根据指定的位置范围切割FASTA文件，并保存到单独的文件中。

    :param input_file: 输入的FASTA文件路径
    :param sequences: 序列列表
    :param ranges: 切割范围列表，例如 [(1, 200), (201, 400)]
    """
    base_name = input_file.rsplit(".", 1)[0]
    
    for start, end in ranges:
        output_file = f"{base_name}_{start}-{end}.fasta"
        with open(output_file, "w") as out_f:
            for record in sequences:
                # 提取指定范围的序列
                sub_seq = record.seq[start-1:end]
                sub_record = record[:0]  # 创建一个空记录
                sub_record.seq = sub_seq  # 设置序列
                sub_record.id = record.id  # 保留原ID
                sub_record.description = f"{record.description} [{start}-{end}]"  # 添加范围信息
                # 写入到输出文件
                SeqIO.write(sub_record, out_f, "fasta")
        print(f"Generated {output_file} for range {start}-{end}")

# 输入文件
input_file = "C:/Users/victo/Desktop/最终版本ALN的编码区.fasta"

# 读取FASTA文件中的序列
sequences = list(SeqIO.parse(input_file, "fasta"))

# 打印每个序列的长度
print_sequence_lengths(sequences)

# 获取用户输入的范围
ranges = get_ranges_from_user()

# 调用函数进行切割
split_fasta(input_file, sequences, ranges)

得到一些切割之后的文件，我们就开始寻找最优模型。
注意！切割之前的原始文件需要保留，我们后续进行 BEAST 需要原始文件。

寻找最优模型

在这里，有许多软件可以帮助寻找最优模型，我们现在使用 iqtree2，因为这个软件经过了更新迭代之后，获得了更强、更广泛的功能，属于建模的万金油。所以我们选择使用它来寻找：
iqtree 的下载地址如下： IQ-TREE: Efficient phylogenomic software by maximum likelihood (iqtree.org)

在这里，我推荐使用 Linux版本，因为操作相对来说比较简单。点击 ALL Downloads，可以查看所有的版本。


下载安装包之后，放到 Linux 环境下，进行解压：

tar -xzf 您的压缩包.gz

运行相当简单，在目录中寻找 iqtree/iqtree-2.3.4-Linux-intel/bin/iqtree2，然后直接运行它输入参数即可。
在运行之前，建议对已有的 fasta 文件进行删减，只保留你需要进行分析的位点，因为如果位点太多，会导致建树非常慢，而且模型寻找也慢。

现在开始运行 iqtree：

#!/bin/bash

# 定义输入文件和输出前缀
declare -A FILES # 下面更改为你切割出来的每一个文件
FILES["编码区"]="/home/luolintao/Mydata/10K/最终版本ALN的编码区.fasta"
FILES["D环_"]="/home/luolintao/Mydata/10K/最终版本ALN的D环.fasta"
# 设置IQ-TREE可执行文件的路径
IQTREE_EXEC="/home/luolintao/iqtree/iqtree-2.3.4-Linux-intel/bin/iqtree2"

# 遍历每个文件并运行IQ-TREE的ModelFinder功能
for NAME in "${!FILES[@]}"; do
    INPUT_FILE="${FILES[$NAME]}"
    OUTPUT_PREFIX="/home/luolintao/Mydata/10K/${NAME}_model_selection"

    echo "Running IQ-TREE ModelFinder for $INPUT_FILE..."

    nohup $IQTREE_EXEC -s "$INPUT_FILE" -m MF -nt AUTO -pre "$OUTPUT_PREFIX" > "$OUTPUT_PREFIX.log" 2>&1 &

    # 检查命令是否成功启动
    if [ $? -eq 0 ]; then
        echo "IQ-TREE ModelFinder for $NAME successfully started."
        echo "Output files are prefixed with $OUTPUT_PREFIX"
    else
        echo "IQ-TREE ModelFinder for $NAME encountered an error." >&2
        exit 1
    fi
done

echo "All IQ-TREE ModelFinder tasks have been started."

运行完成之后，我们找到后缀为 .iqtree 的文件，文件的内容大致如下：

这里我们观察：

  1. AIC (Akaike Information Criterion)：用于模型选择，较低的AIC值表示较好的模型拟合。
  2. AICc (Corrected Akaike Information Criterion)：对于小样本量的修正AIC。
  3. BIC (Bayesian Information Criterion)：另一个用于模型选择的准则，也较低的BIC值表示较好的模型。

根据表格数据：

  • AIC 最低：GTR+F+I+G4 (-22875.459, 46120.919)
  • AICc 最低：GTR+F+I+G4 (-22875.459, 46125.444)
  • BIC 最低：TIM3e+I+G4 (-23625.409, 48986.330)

我们选择的样本会影响到最优模型的构建，在这里我们选择最低的值。这里出现了3个最低，我们需要选择一个。我们选择 GTR+F+I+G4模型。

另外的分割序列也如法炮制，记住我们选择的最优模型。
例如，在我这里，高变区我认为 TIM2+F+I 是最优的模型。

  • TIM2+F+I (-1083.148, 2442.379)

使用 BEAUTi 制作参数文件

现在，使用我们一开始的原始文件。将原始文件导入 BEAUTi 软件，
在导入之前，我们需要先把 ALN 文件转为 NEX 文件，转的方式很多，最简单的方法就是直接把 FASTA 导入 MEGA 软件，然后导出来的时候，点击 NEX 就可以了。

NEX 文件分区


在这里，我们可以看到 NEX 文件的末尾什么都没有，但是我们需要增加分区，分区依然按照我们之前分区进行。
例如：

begin assumptions;
charset DLOOP = 1-599;
charset CODING = 600-16594;
end;

将这段内容修改为自己的分区，追加到 nex 文件中去。再把新的 nex 文件导入 BEAUti。
